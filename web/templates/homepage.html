Why Weft?

The default Power BI deploy is destructive — it replaces your model and wipes every partition your refresh policy materialized. Weft does a three-way diff and only touches what changed. Read the rationale →

Built for ops

Five auth modes (cert + secret + interactive). YAML config with per-environment profiles. Hooks at six lifecycle phases. History-loss gate that refuses to silently drop years of data. See the hooks guide →

Open, MIT, .NET 10

Cross-platform CLI plus a Weft.Core NuGet for embedding. GitHub Actions, TeamCity, Octopus templates included. Browse the source →
